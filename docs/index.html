Architecture

Librairie de design tokens agnostique de tout framework, au format DTCG ($value / $type / $description). Ce repo ne contient que les tokens — pas de composants UI.

Les deux couches du milieu portent tout le système. Brand rend un projet rebrandable en éditant une ligne. Sémantique impose un contrat uniforme : les 6 rôles exposent les mêmes 9 slots, mappés sur les mêmes crans — c'est ce qui rend <Button color="danger"> aussi lisible que <Button color="primary">, sans le vérifier à la main.

Les 3 règles de layering

  • Un token de composant référence un rôle sémantique, jamais une couche plus basse.
  • Un rôle sémantique de couleur référence la couche brand, jamais un primitive.
  • Un token brand référence un primitive, jamais une valeur brute.

Elles sont mécaniquement vérifiables : un grep suffit à prouver qu'aucun composant ne pointe sur une palette.

La chaîne de résolution

C'est la chose à comprendre. Un token de composant ne connaît ni les palettes ni les hex : il ne connaît qu'un rôle. Chaque cran descend d'une couche.

  1. component color.button.filled.primary.default.background
  2. semantic color.role.primary.main
  3. brand color.brand.primary.600
  4. primitive color.violet.600 #9500ee
